<div><p><em>Welcome to YUI Weekly, the weekly roundup of news and announcements from the YUI team and community. If you have any interesting demos or links you&rsquo;d like to share, feel free to leave a comment below.</em></p>
<ul><li>
<p>Hot of the release of last week&rsquo;s YUI 3.12.0 GA, this week saw the release of <strong>YUI 3.13.0 Beta 1</strong>. As noted in the <a href="http://www.yuiblog.com/blog/2013/09/04/yui-3-13-0-beta-1-released/">announcement</a>, the primary focus of this release is to provide a build for developers to test a new <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Content_Editable">content editable feature</a> in <a href="http://stage.yuilibrary.com/yui/docs/editor/">Editor</a>. Also included in this release are updates to <a href="http://stage.yuilibrary.com/yui/docs/datatable/">DataTable</a>, <a href="http://stage.yuilibrary.com/yui/docs/history/">History</a>, <a href="http://stage.yuilibrary.com/yui/docs/api/modules/node-scroll-info.html">ScrollInfo</a>, and <a href="http://stage.yuilibrary.com/yui/docs/widget/">Widget</a>. To see all the specific changes, check out the <a href="https://gist.github.com/triptych/6430650">YUI 3.13.0 Beta 1 Change History Rollup</a>. Happy testing!</p>
</li>
<li>
<p>At this week&rsquo;s <a href="https://github.com/yui/yui3/wiki/Open-Roundtable">Open Roundtable</a>, discussion topics included <a href="http://www.yuiblog.com/blog/2013/09/04/yui-3-13-0-beta-1-released/">YUI 3.13.0 Beta 1</a>, an inline Editor demo, discussion about upcoming Widget and Button updates, and YUIConf. You can find a video of that Hangout <a href="http://www.youtube.com/watch?v=Fks_KWzwU30">here</a>.</p>
</li>
<li>
<p>Just a friendly reminder, a few weeks ago the project <a href="http://www.yuiblog.com/blog/2013/08/21/announcing-the-new-yui-community-forums/">announced</a> plans to migrate from the YUI Forums to <a href="https://groups.google.com/forum/#!forum/yui-support">yui-support</a>. Please subscribe to the new group if you haven&rsquo;t already, and begin posting any support questions there instead of the forum. Thanks!</p>
</li>
