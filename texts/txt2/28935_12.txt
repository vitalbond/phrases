<ul><li>
<p><a href="https://github.com/yui/yui3/wiki/Development-Schedule#sprint-8-current">Sprint 8</a> continues this week as we approach Sunday&rsquo;s pull request deadline for the upcoming 3.11 release. Thanks for all the testing and feedback following last week&rsquo;s <a href="http://www.yuiblog.com/blog/2013/06/19/3-11pr1-released/">3.11pr1 release</a>, especially all the discussion in <a href="https://github.com/yui/yui3/pull/917">Pull #917</a>. Next up in this dev cycle is a pull request deadline on July 9th before the 3.11 release window opens up on July 16th.</p>
</li>
<li>
<p>We&rsquo;d like to extend a warm YUI welcome to Clarence Leung, who joined the team this week. You might remember Clarence from his <strong>awesome work</strong> last summer as an intern, and we enjoyed having him around so much we decided to bring him on full-time! He joins us from Montreal, Canada where he graduated from McGill University with degrees in Computer Science and Biology, and you can find him on Twitter as <a href="http://twitter.com/clarler">@clarler</a>, Github as <a href="http://github.com/clarle">@clarle</a>, and IRC as &ldquo;clarle&rdquo;.</p>
</li>
<li>
<p>While YUI contributors have been using <a href="http://benchmarkjs.com/">Benchmark.js</a> as a performance testing tool for some time, it was never included in the source tree and available locally for use within tests. Well that changed this week when it was <a href="https://github.com/yui/yui3/pull/918">pulled in</a>, and a huge thanks to Juan Dopazo (<a href="https://twitter.com/juandopazo">@juandopazo</a>) for the <a href="http://yuilibrary.com/yui/docs/promise/">Y.Promise</a> performance tests <a href="https://github.com/yui/yui3/pull/926">he contributed</a>. If you are interested in helping YUI get faster by contributing tests, check out the <a href="http://benchmarkjs.com/docs">Benchmark.js documentation</a> and <a href="https://github.com/yui/yui3/tree/master/src/promise/tests/performance">some examples</a> of <a href="https://github.com/yui/yui3/tree/master/src/app/tests/performance">perf tests</a> currently in the library.</p>
</li>
